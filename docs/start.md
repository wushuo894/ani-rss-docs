# 快速开始

如果你是第一次接触 `ani-rss`，建议按下面的顺序完成：

1. 先选一个适合自己的部署方式
2. 启动程序并登录后台
3. 配置下载器、保存路径、代理等基础项
4. 添加一个测试订阅，确认可以正常下载
5. 再继续折腾重命名、通知、自动刮削等进阶功能

## 第一步：选择部署方式

不同部署方式适合的人群不同，可以先按使用场景选择：

| 场景             | 推荐方式                          | 适合人群                       |
|----------------|-------------------------------|----------------------------|
| 日常主力机是 Linux   | [Linux 部署](/deploy/linux)     | Linux 用户                   |
| 想快速开始、后续也方便升级  | [Docker 部署](/deploy/docker)   | 已安装 Docker / NAS / 家庭服务器用户 |
| 日常主力机是 macOS   | [macOS 部署](/deploy/macos)     | macOS 用户                   |
| 日常主力机是 Windows | [Windows 部署](/deploy/windows) | Windows 用户                 |
| 使用群晖 NAS       | [群晖 NAS](/deploy/dsm)         | DSM 用户                     |

::: tip 推荐选择
大多数用户优先推荐 [Docker 部署](/deploy/docker)，环境隔离更清晰，后续升级和迁移也更方便。
:::

## 第二步：完成基础准备

开始部署前，建议先准备好这些信息：

- 一个可用的下载器：推荐 `qBittorrent`
- 一个用于保存番剧的目录
- 可正常访问外部资源站点的网络环境
- 如果你使用 Docker，请提前确认 `ani-rss` 与下载器的挂载路径设计

如果你还没配置下载器，可以先看：

- [下载设置](/config/download)
- [qBittorrent 设置](/config/download/qbittorrent)

如果你的网络访问 `Mikan`、`Bangumi`、`TMDB`、`Telegram` 不稳定，可以再看：

- [代理设置](/config/proxy)

## 第三步：启动并登录

完成部署后，在浏览器打开程序地址：

- 默认一般为 `http://127.0.0.1:7789`
- Docker / 反代 / 局域网环境请按你自己的实际地址访问

默认账号：

- 用户名：`admin`
- 密码：`admin`

::: warning 首次登录后建议立刻做的事

1. 前往 [登录设置](/config/login) 修改默认账号密码
2. 如果服务需要被局域网或公网访问，顺手检查 `IP 校验`、`IP 白名单`、`信任的反代IP`
3. 如果后续要和第三方工具联动，记得保存好 `API Key`

:::

## 第四步：先完成这几项关键配置

真正影响“能不能正常追番”的，通常是下面几项：

### 1. 下载器

先在 [下载设置](/config/download) 中配置：

- 下载工具类型
- 地址
- 用户名
- 密码
- 保存位置

如果你使用 Docker，请特别注意：

- `ani-rss` 与下载器的挂载路径必须保持一致
- 否则自动跳过、自动删除、重命名等功能很容易异常

### 2. 保存路径与命名

建议至少了解这些页面：

- [下载设置](/config/download)
- [重命名设置](/config/basic/rename)
- [RSS 设置](/config/basic/rss)

如果你后续要让 Emby / Jellyfin 更好识别，建议尽量使用默认或接近默认的重命名模板。

### 3. 代理与网络

如果你已经遇到下面这些现象：

- RSS 获取超时
- SSL 握手异常
- DNS 解析失败
- Bangumi / TMDB 请求失败

优先检查：

- [代理设置](/config/proxy)
- [常见问题](/faq#ssl-error)

## 第五步：添加一个测试订阅

第一次使用时，不要一上来就导入很多订阅，建议先用一个测试番剧验证整条链路。

推荐顺序：

1. 打开 [添加订阅](/add-rss)
2. 填写标题、主 RSS、保存位置等基础信息
3. 保持其他高级选项先用默认值
4. 保存后观察是否能正常拉取、识别、推送到下载器

如果集数识别不对，再看：

- [自定义集数匹配规则](/custom-episode)
- [添加订阅配置](/config/basic/add)

## 第六步：确认是否已经配置成功

只要下面几项都正常，基本就算跑通了：

- 页面里能看到订阅条目
- RSS 能正常拉取，不报网络错误
- 下载器里能成功出现任务
- 下载完成后文件路径符合预期
- 如果开启了重命名，文件名符合你的模板规则

如果某一步不正常，建议优先排查：

- [常见问题](/faq)
- [下载设置](/config/download)
- [RSS 设置](/config/basic/rss)
- [重命名设置](/config/basic/rename)

## 第七步：再启用进阶功能

当基础下载流程没问题后，再逐步开启这些能力会更稳：

- [通知](/config/notification)
- [页面设置](/config/basic/page)
- [Trackers](/config/basic/trackers)
- [其他配置](/config/basic/other)
- [忘记密码](/change-the-password)

::: tip 使用建议
建议一次只调整一到两个功能，改完就用一个测试订阅验证。这样出问题时，最容易定位到底是哪一项配置造成的。
:::

## 常见新手路线

如果你不知道该从哪里开始，可以直接按这个最小流程来：

1. 选择 [Docker 部署](/deploy/docker)
2. 配置 [下载设置](/config/download)
3. 修改 [登录设置](/config/login)
4. 添加一个 [测试订阅](/add-rss)
5. 确认正常后，再看 [重命名设置](/config/basic/rename) 和 [通知](/config/notification)

这样通常是最不容易踩坑的一条路线。
